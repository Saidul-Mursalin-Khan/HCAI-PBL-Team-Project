Now assume we start with zero expert labels and may only query the expert a limited number of times. We compare four strategies for choosing which training points to query: random sampling (control), uncertainty sampling (query the AI's least confident points), margin sampling (query the smallest gap between the AI's top-2 classes), and a hybrid of random + uncertainty. After each query batch we retrain the Task 3 deferral gate on just the queried points and evaluate on the held-out test set.

At very small budgets, pure uncertainty/margin sampling can actually collapse to an "always defer" policy: because it only ever queries the AI's hardest points, the gate never sees an easy, AI-correct example to contrast against, so it can't learn when deferring does not help. The hybrid strategy avoids this failure mode by keeping some random exploration, at the cost of using its query budget slightly less efficiently once the budget is larger.
